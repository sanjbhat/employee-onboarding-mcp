Your First Pull Request

Make your first contribution to the codebase by submitting a pull request. This helps you get familiar with our development workflow.

What You Need to Do:

  1. Set Up Development Environment:
    • Clone the team's main repository from Azure DevOps
    • Install required development tools (VS Code, Git, Node.js/Python/.NET as needed)
    • Run the project locally to ensure everything works
  2. Pick a Beginner-Friendly Task:
    • Look for issues tagged with "good-first-issue" or "beginner-friendly"
    • Common first tasks: update documentation, fix typos, add unit tests
    • Ask your onboarding buddy for task recommendations
  3. Create Your Branch:
    • Create a new branch: git checkout -b feature/your-name-first-pr
    • Follow our branch naming conventions
    • Make your changes in small, focused commits
  4. Submit Pull Request:
    • Push your branch to Azure DevOps
    • Create a pull request with a clear title and description
    • Add your onboarding buddy as a reviewer
    • Link to the work item/issue you're addressing
  5. Address Feedback:
    • Respond to code review comments promptly
    • Make requested changes and push updates
    • Ask questions if feedback is unclear

  • Add input validation to a form or API endpoint
